## 已知问题 / 注意事项

1. **Audio 文件夹混乱**：`Audio/` 目录下同时存在 `.rxdata` 数据文件（Actors、Animations、Armors 等）和音频子目录（BGM/SE 等），这些 `.rxdata` 文件似乎是错误放置的副本，实际数据在 `Data/` 目录下。

2. **编码问题**：项目创建于2007年，文本文件（如 `宠物小精灵游戏介绍.txt`）使用 GBK/GB2312 编码，在非中文环境下可能乱码。

3. **地图数量巨大**：354张地图（Map001-Map354），`Map354.rxdata` 特别大（264KB），可能是大地图或复杂场景。

4. **缺少 RTP 依赖**：`Game.ini` 中 RTP1/RTP2/RTP3 均为空，说明此游戏不依赖标准 RTP，所有资源已自包含。

5. **音频格式混用**：BGM 混用 MP3/WMA/MID 三种格式，在不同系统上 MIDI 播放效果可能不同；SE 混用 OGG/WAV/MP3。

6. **旧版引擎**：RGSS104E 是较老版本，在 Windows 10/11 上可能需要兼容性设置才能运行。

## 开发/维护指引

### 编辑器
- 使用 **RPG Maker XP** 打开 `Game.rxproj` 即可编辑地图、事件、数据库
- 脚本编辑：在 RPG Maker XP 中按 F11 打开脚本编辑器，或直接解析 `Data/Scripts.rxdata`

### 数据解包/打包
如果需要解析或修改 `.rxdata` 文件，可以使用以下 Ruby 脚本：
```ruby
# 读取 .rxdata 文件
data = File.open("Data/Actors.rxdata", "rb") { |f| Marshal.load(f) }
# 修改后保存
File.open("Data/Actors.rxdata", "wb") { |f| Marshal.dump(data, f) }
```

### 资源替换规则
- **Battlers**（战斗图）：命名需与游戏内敌人/精灵编号对应，支持 `_b`（背面）、`_f`（正面）、`_s`（闪光）后缀
- **Characters**（行走图）：标准 RPG Maker 4×4 帧格式，每帧大小 = 图宽/4 × 图高/4
- **Icons**（图标）：24×24 像素 PNG 格式
- **Pictures**（大图鉴）：精灵正视图，含 `_big` 版本用于详细信息界面

### 修改脚本注意事项
- 脚本存储在 `Data/Scripts.rxdata`，是 Ruby 对象数组（每个元素为一个脚本页）
- 脚本结构：[页号, 页名, ZLib压缩后的脚本内容]
- 大量自定义脚本实现了精灵捕捉、进化、联网等原作不具备的功能
- 修改前务必备份 Scripts.rxdata

### 运行游戏
1. 确保 `RGSS104E.dll` 在游戏目录下
2. 直接运行 `Game.exe`
3. 如闪退，尝试以管理员权限运行或设置 Windows XP SP3 兼容模式
